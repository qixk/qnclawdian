## 一、Roo 框架核心原则（来自 video-analysis-agent-long-running.md）

| # | 原则 | 实施方式 |
|---|------|---------|
| 1 | **每轮全新上下文** | 每个 Agent 完成一个 User Story 后，上下文清零重新开始 |
| 2 | **三件持久化文件** | `prd.json`（任务清单）+ `lessons.md`（经验日志）+ Git 提交 |
| 3 | **每轮强制测试** | `tsc --noEmit` + `esbuild` 必须通过才能标记完成 |
| 4 | **验收标准写死** | 每个 User Story 有精确的 pass/fail 标准 |
| 5 | **任务拆成最小单元** | 每个 Story 一轮（一次 Agent 调用）能完成 |
| 6 | **经验日志持续积累** | 踩过的坑写入 lessons.md，下一轮自动加载 |

## 四、执行流程（Roo 循环）

```
┌──────────────────────────────────────────────┐
│                 Roo 循环                      │
│                                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 1: 龙龙读取 prd.json              │ │
│  │         找到下一个 status=pending 任务   │ │
│  └─────────────┬───────────────────────────┘ │
│                ↓                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 2: 龙龙分配任务给负责 Agent        │ │
│  │         发送 sessions_send              │ │
│  └─────────────┬───────────────────────────┘ │
│                ↓                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 3: 大拿马执行编码                  │ │
│  │         读取 lessons.md 避免重复踩坑     │ │
│  └─────────────┬───────────────────────────┘ │
│                ↓                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 4: 强制测试                        │ │
│  │   ① tsc --noEmit（类型检查）            │ │
│  │   ② npm run build（构建验证）           │ │
│  │   ③ 验收标准逐项检查                    │ │
│  └─────────────┬───────────────────────────┘ │
│                ↓                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 5: 测试通过？                      │ │
│  │   ✅ → git commit + 标记 done           │ │
│  │   ❌ → 写入 lessons.md + 修复重试       │ │
│  └─────────────┬───────────────────────────┘ │
│                ↓                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 6: 小秘记录本轮经验                │ │
│  │         小美/小马哥做验收测试            │ │
│  │         小法做安全审查                   │ │
│  │         小财统计 Token 消耗              │ │
│  └─────────────┬───────────────────────────┘ │
│                ↓                              │
│  ┌─────────────────────────────────────────┐ │
│  │ Step 7: 龙龙更新 prd.json 进度          │ │
│  │         还有任务？→ 回到 Step 1          │ │
│  │         全部完成？→ 生成最终报告         │ │
│  └─────────────────────────────────────────┘ │
└──────────────────────────────────────────────┘
```

## 五、持久化文件

### 1. prd.json（任务清单）
位置: `projects/qnclawdian/prd.json`

### 2. lessons.md（经验日志）
位置: `projects/qnclawdian/docs/LESSONS.md`

初始内容（来自 M1 测试报告踩坑）：
- **Lesson 1**: 编辑 TypeScript 后必须 `tsc --noEmit`，不要跳过
- **Lesson 2**: 使用 `textContent` 防 XSS，但需要 MarkdownRenderer 做富文本渲染
- **Lesson 3**: Ollama gemma4:26b 返回 thinking 字段，类型定义必须包含
- **Lesson 4**: 模型下拉框不能用空字符串做分组标题，要用 optgroup
- **Lesson 5**: 流式请求必须有超时保护（120s），否则卡死无提示
- **Lesson 6**: Obsidian CSS 变量必须用原生变量，不能硬编码颜色值
- **Lesson 7**: compositionstart/compositionend 处理中文输入法，不能漏

### 3. Git 提交记录
每个 Story 完成后 git commit，消息格式: `feat(M2-S01): 启用 MarkdownRenderer`

## 六、验收门控（Quality Gate）

每轮必须通过以下检查才能标记 done：

| 检查项 | 命令/方法 | 通过标准 |
|--------|----------|---------|
| TypeScript 类型检查 | `npx tsc --noEmit` | 零错误 |
| 构建成功 | `npm run build` | 输出 main.js + styles.css |
| 验收标准 | 逐项对照 Story 验收标准 | 100% 满足 |
| 安全扫描 | 小法检查 innerHTML/eval/XSS | 无安全问题 |
| 代码审查 | 小美/小马哥 UI/API 验收 | 通过 |

## 七、风险预案

| 风险 | 应对 |
|------|------|
| Ollama 内存不足 | 切换到 gemma3:4b 或使用 Gateway 路由 |
| Agent 上下文溢出 | 严格执行"每轮全新上下文"，不积累 |
| 构建失败循环 | 3次修复失败 → 回退到上一个 git commit |
| 网络断开 | 所有文件本地持久化，恢复后继续 |
| 凯哥紧急任务 | 保存当前进度到 prd.json，随时可暂停恢复 |
